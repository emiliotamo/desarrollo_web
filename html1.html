DESCRIPCIÓN DE REQUISITOS

Requisitos funcionales

  • Catalogo: Los usuarios podrán ver todos los productos organizados por categorías.

  • Carrito: Permite a los usuarios agregar, revisar y modificar productos antes de la compra. Facilita la gestión de los artículos seleccionados

  • Cuentas de ususario:Permiten crear un perfil, guardar información personal y ver el historial de compras, agilizando futuras transacciones.

  • Control de pedidos: Permite a los usuarios y administradores ver y gestionar el estado de las órdenes en proceso, completadas o canceladas

  • Opciones de pago: Ofrece diferentes métodos de pago (tarjetas, PayPal, transferencias, etc.), proporcionando flexibilidad al cliente.

  • Envío: Permite configurar y elegir métodos de envío, calculando costos y tiempos de entrega según la ubicación.

  • Seguimiento de envío: Proporciona a los usuarios información en tiempo real sobre el estado y la ubicación de sus pedidos en tránsito.

  • Notificaciones:Envía alertas al usuario sobre cambios en pedidos, envíos, promociones o actualizaciones importantes del sitio.

Requisitos no funcionales

  • Rendimiento: a interfaz debe ser fácil de usar e intuitiva, con una navegación fluida entre las secciones del catálogo, carrito de compras y gestión de cuentas

  • Escalabilidad:La infraestructura debe ser capaz de crecer y adaptarse para soportar un aumento en la cantidad de usuarios, productos y transacciones sin afectar su rendimiento.

  • Seguridad:El sitio debe garantizar la protección de los datos personales y de pago de los usuarios mediante protocolos de seguridad

  • Usabilidad: La interfaz debe ser intuitiva, fácil de navegar y accesible para todo tipo de usuarios, incluyendo aquellos con discapacidades.

  • Disponibilidad:a plataforma debe estar disponible 24/7, con un mínimo de tiempo de inactividad durante actualizaciones o mantenimiento.

  • Compatibilidad:El sitio debe ser compatible con distintos navegadores (Chrome, Firefox, Safari, etc.) y sistemas operativos

  • Mantenimiento:El código de la aplicación debe estar bien documentado para facilitar futuras actualizaciones o correcciones.

DISEÑO DE LA ARQUITECTURA

  1. Frontend desarrollado en React para una interfaz ágil y responsiva, adaptada a móviles y tablets.

  2. Backend con Node.js y una API REST para manejar operaciones de usuario, carrito y pagos.

  3. Base de datos MySql para gestionar los productos, usuarios y pedidos.

  4. Integración de la pasarela de pago Stripe para tarjetas y Coinbase Commerce para criptomonedas.

IMPLEMENTACIÓN

  1. Interfaz de Usuario
    • Diseño Responsive: Optimización para dispositivos móviles y desktop.
    • Menú de navegación claro: Categorías del catálogo bien organizadas
    • Motor de búsqueda avanzado: Filtros por precio, popularidad, disponibilidad, etc.


  2. Carrito de Compras Funcional:
    • Actualización en tiempo real: Reflejar los cambios en el carrito sin recargar la página.
    • Guardar el carrito: Opciones para guardar productos para más tarde o continuar con el carrito en visitas futuras.
    • Aplicar cupones y códigos de descuento: Validación en tiempo real de descuentos.
